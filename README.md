# KatoVape

Демо-витрины для вейп-шопа KatoVape. Магазин продаёт украинские устройства и жидкости в Польше и ведёт наличие через закрытые Telegram-каналы по городам. Вместо поста «АКТУАЛЬНОЕ НАЛИЧИЕ» с галочками тут сайт и мини-приложение, которые сами читают базу и показывают остатки, цены, вкусы и характеристики.

Собрано три стиля, в каждом пара «сайт + мини-приложение для Telegram». Всё статическое, без бэкенда: страницы тянут JSON и рисуют каталог.

## Что внутри

- **Неон**: тёмная тема, кислотный акцент, логотип-кот. `demos/neon/`
- **Фрост**: светлое морозное стекло, синий градиент. `demos/frost/`
- **Стрит**: жёлто-синяя стикер-бомба со штампами. `demos/street/`

Точка входа: `index.html` (хаб со ссылками на все шесть страниц).

## Возможности

- Выбор города (Катовице, Гливице, Варшава). У каждого свой файл наличия, свои остатки и свой логотип.
- Корзина с автозаказом: по кнопке открывается Telegram с уже набранным сообщением (товары, город, сумма, самовывоз).
- Бронь отдельной позиции.
- Три языка: русский, украинский, польский, с переводом даже названий вкусов.
- Переключатель светлой и тёмной темы.
- Характеристики товара, вкусы с остатком по каждому, поиск, метка новинок, возрастной гейт 18+.

## Как устроено

Вся логика в `shared/core.js`, страницы отвечают только за вёрстку. Каталог лежит в `data/`:

- `data/products.json`: Катовице (главный, полный список) плюс список городов.
- `data/gliwice.json`, `data/warszawa.json`: наличие остальных городов.
- `data/photos/`: фото товаров (`<id>.jpg`) и логотипы городов (`cat-<city>.png`).

Поменял число в JSON, обновил страницу, и маркеры с остатками изменились. В боевой версии базу будет наполнять админка или гугл-таблица, контракт данных тот же.

## Запуск

Любой статический сервер из корня проекта:

```
python -m http.server 8090
```

Дальше открыть `http://localhost:8090`. Мини-приложения удобно смотреть в мобильном виде браузера; внутри Telegram нижняя кнопка становится нативной.

## Статус

Демо. Цены и часть вкусов прикидочные, пока не залиты реальные данные из каналов. Дальше по плану: выбор стиля заказчиком, админка с гугл-таблицей, бот и хостинг.
