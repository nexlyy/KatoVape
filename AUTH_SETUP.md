# KatoVape: подключение авторизации

Вход/регистрация работают на Supabase. Пароли хешируются самим Supabase (bcrypt),
сессии на JWT, данные шифруются в БД и передаются по TLS. Токен бота и service_role
ключ живут только на сервере (в секретах Edge Function) и в браузер не попадают.

Пока `shared/config.js` пустой, сайт работает как раньше (гостевой режим). Как только
впишешь ключи, вход включится сам.

## 1. Создать проект Supabase

1. Заведи проект на https://supabase.com (бесплатного тарифа хватит).
2. Возьми `Project Settings → API`: **Project URL** и ключ **anon/public**.

## 2. Накатить схему

В Supabase `SQL Editor` вставь и выполни `supabase/migrations/0001_auth.sql`.
Появится таблица `profiles`, триггер автосоздания профиля, RLS и две функции
(`login_availability`, `resolve_login`).

Либо через CLI:
```
supabase link --project-ref <ref>
supabase db push
```

## 3. Настройки Auth

`Authentication → Providers → Email`: включи Email.

- Хочешь регистрацию без почты (по логину или телефону) — выключи
  **Confirm email**. Тогда аккаунт активен сразу.
- Оставляешь подтверждение почты включённым — тогда регистрация с реальной почтой
  потребует перехода по ссылке из письма (форма это покажет). Вход по логину/телефону
  использует синтетический адрес и подтверждения не требует.

## 4. Edge Function для Telegram

Нужна, потому что подпись Telegram проверяется секретным токеном бота, которому
нельзя быть в браузере.

```
supabase functions deploy telegram-auth --no-verify-jwt
supabase secrets set TELEGRAM_BOT_TOKEN=<токен_бота_от_BotFather>
```

`SUPABASE_URL`, `SUPABASE_SERVICE_ROLE_KEY`, `SUPABASE_ANON_KEY` в функции доступны сами.
Флаг `--no-verify-jwt` обязателен: функцию зовут ещё не залогиненные пользователи.

### Кнопка «Войти через Telegram» на сайте

1. У @BotFather: `/setdomain` → укажи домен сайта (напр. `nexlyy.github.io`).
   Login Widget работает только на заявленном домене.
2. Впиши username бота в `TELEGRAM_BOT` в `config.js`.

### Мини-апп

Отдельно ничего не нужно: при открытии в Telegram человек авторизуется автоматически
по `initData` (та же функция, режим `initdata`).

## 5. Заполнить config.js

```js
window.KV_CONFIG = {
  SUPABASE_URL:      'https://xxxx.supabase.co',
  SUPABASE_ANON_KEY: 'eyJ...',                       // anon/public, светить безопасно
  FUNCTIONS_URL:     'https://xxxx.supabase.co/functions/v1',
  TELEGRAM_BOT:      'ваш_бот_без_собачки'           // только для кнопки на сайте
};
```

## Как устроен вход

- **Логин/почта/телефон + пароль.** Supabase логинит по email, поэтому при регистрации
  без почты заводится синтетический адрес (`u_<логин>@users.katovape.local`), а человек
  входит любым своим идентификатором: функция `resolve_login` находит нужный email.
- **Проверка занятости.** Перед регистрацией `login_availability` подсказывает, что
  логин/почта/телефон уже заняты. Финальная гарантия — уникальные индексы в БД
  (гонки не пройдут).
- **Telegram.** Фронт шлёт данные в `telegram-auth`, она проверяет подпись бот-токеном,
  находит или заводит пользователя и отдаёт одноразовый OTP. Фронт меняет его на сессию.

## Что можно усилить дальше

- Перенести избранное, отзывы и историю заказов из localStorage в таблицы с RLS,
  чтобы они ходили за аккаунтом между устройствами.
- Включить телефонную OTP-верификацию через SMS-провайдера (Twilio и т.п.), если нужен
  вход по коду из СМС, а не по паролю.
- Для особо чувствительных полей — шифрование на уровне колонок (pgsodium/Vault).
